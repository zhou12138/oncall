DATA FLOW SUMMARY:
═════════════════════════════════════════════════════════════════════════════

HTTP Trigger (natural language)
    │
    ├──→ Intent Extraction (LLM)
    │    └──→ Structured: {signal_name, repo, teams_channel, ...}
    │
    └──→ Orchestrator.run()
         │
         ├──→ Step 1: Triage (ADX via MCP)
         │    Output: {verdict, platform_breakdown}
         │
         ├──→ Step 2: WoW (ADX + GitHub via MCP)
         │    Output: {current_count, delta, trend, recent_changes}
         │
         ├──→ Memory.recall(signal_name)
         │    Returns: Similar past incidents (Jaccard similarity)
         │
         ├──→ Step 3: Reason (LLM)
         │    Input: Step 1 + Step 2 + Memory context
         │    Output: {summary, severity, actions}
         │
         ├──→ Memory.record() + Memory.add()
         │    Persist: incidents, patterns, wow_comparisons
         │
         └──→ Teams.send_message() (optional)
              Card: Adaptive Card with Ack/Escalate buttons


STORAGE & PERSISTENCE:
════════════════════════════════════════════════════════════════════════════

Memory (JSON):
  ~/.oncall/memory.json
    ├─ incidents [4000 chars]
    ├─ patterns [2000 chars]
    ├─ runbooks [2000 chars]
    └─ wow_comparisons [1500 chars]
    Total: 10,000 chars max
    Auto-eviction: Oldest entries first

Workspace (Markdown + JSON):
  ~/.oncall/workspaces/{name}/
    ├─ soul.md (project identity + escalation rules)
    ├─ memory.md (auto-updated by agent)
    └─ config.json (workspace-specific overrides)

Active Workspace:
  ~/.oncall/active_workspace (single file, workspace name)

Credentials:
  OS Keyring (preferred) → ~/.oncall/copilot_credentials.json (fallback)
  ~/.oncall/.env (secrets, loaded first)
  ~/.oncall/config.json (config)


ERROR HANDLING:
═════════════════════════════════════════════════════════════════════════════

OncallError (base)
  ├─ TriageError ─────────→ Step 1 failure (ADX query, parsing)
  ├─ WoWError ────────────→ Step 2 failure (metrics, GitHub)
  ├─ ReasoningError ──────→ Step 3 failure (LLM, Teams send)
  └─ MCPError ────────────→ Transport-level MCP failure

Per-step wrapping: Allows orchestrator to aggregate errors without losing context
Retry logic: MCP client retries transient errors (5xx, timeout) 3 times (1s/2s/4s)
